# Google Places API クローラー仕様書

## 1. 概要

Google Places APIを利用して日本の飲食店情報を収集し、Amplifyで構築されたデータベースに格納するクローラーの仕様を定める。

## 2. 目的

-   アプリケーションで利用する飲食店データを充実させる。
-   定期的（例: 週次）に実行し、情報の鮮度を保つ。

## 3. データソース

-   **API**: Google Places API
    -   **エンドポイント**: Nearby Search, Place Details
-   **対象地域**: 日本全国の主要都市から開始し、順次拡大する。
-   **対象カテゴリ**: `restaurant` （飲食店）

## 4. データ構造の比較とマッピング

## 5. クローラーの実装方針

-   **言語**: TypeScript
-   **実行環境**: `scripts` ディレクトリ内にクローラー用のスクリプト (`crawl.ts`) を作成し、`npx tsx scripts/crawl.ts` のように実行する。
-   **ライブラリ**:
    -   `@googlemaps/google-maps-services-js`: Google Places APIを叩くための公式クライアントライブラリ。
    -   `aws-amplify/data`: Amplifyのデータベースを操作するためのライブラリ。
    -   `@aws-sdk/client-bedrock-runtime`: Bedrock APIを叩くためのSDK。
-   **設定**:
    -   Google Places APIのAPIキーは `.env.local` ファイルで管理する。
    -   検索対象の都市リストは、スクリプト内の定数配列として定義する。

## 6. 処理フロー

1.  **初期設定**:
    -   `.env.local` からGoogle Places APIキーを読み込む。
    -   Amplifyクライアントを初期化する。
    -   検索対象の都市リストを定義する。

2.  **都市ごとにループ**:
    -   各都市の中心座標を基点に、Google Places APIのNearby Searchを実行する。
        -   `type: 'restaurant'`
        -   `radius: 5000` (5km)
        -   `language: 'ja'`
    -   検索結果が複数ページにわたる場合(`next_page_token`)は、全てのページを取得する。

3.  **店舗ごとにループ**:
    -   取得した店舗リスト（`Place`オブジェクトの配列）をループ処理する。
    -   **重複チェック**: `place_id` をキーにして、データベースに既に存在するか確認する。存在する場合はスキップする。
    -   **詳細情報取得**: `place_id` を使ってPlace Details APIを叩き、`opening_hours`, `reviews`, `editorial_summary` などの詳細情報を取得する。
    -   **データ整形**:
        -   上記のマッピング表に従い、APIレスポンスを `Restaurant` モデルの形式に変換する。
        -   `ambience`, `keywords` の生成のために、レビューテキストなどをBedrockに渡し、キーワード抽出を依頼する。
    -   **データベース保存**:
        -   整形したデータをAmplifyのData Clientを使ってデータベースに保存する (`client.models.Restaurant.create(...)`)。

4.  **レートリミットとエラー処理**:
    -   Google Places APIの利用制限に抵触しないよう、リクエスト間に適切なウェイトを入れる。
    -   APIリクエストやDB保存でエラーが発生した場合は、エラー内容をログに出力し、処理を継続する。

## 7. 実行方法

1.  `.env.local` ファイルに以下の環境変数を設定する。
    ```
    GOOGLE_PLACES_API_KEY=YOUR_API_KEY
    ```
2.  必要なライブラリをインストールする。
    ```bash
    npm install @googlemaps/google-maps-services-js
    ```
3.  以下のコマンドでクローラーを実行する。
    ```bash
    npx tsx scripts/crawl.ts
    ```
